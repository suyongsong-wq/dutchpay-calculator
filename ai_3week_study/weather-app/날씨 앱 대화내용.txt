[ 과제 ]
----------------------------------------------------------
OpenWeatherMap API를 활용해서 날씨 앱을 만들기.
- 도시 이름을 입력하면 현재 날씨, 온도, 습도, 날씨 아이콘 등을 표시
- units=metric 파라미터로 섭씨 온도 받기
- (보너스 +5점) 사용자 위치정보를 자동으로 감지하는 기능

[ 팁 ]
- API Key 발급 후 활성화까지 최대 2시간 걸릴 수 있으니 미리 발급
- units=metric 으로 섭씨 온도 수신
- 개발자도구 Network 탭에서 API 응답 직접 확인하며 개발

3. API 키 발급
==========================================================
- AI가 직접 계정 가입/키 발급은 불가 (이메일 인증·약관 동의는 본인만 가능)
- OpenWeatherMap 회원가입 페이지를 브라우저로 열어 안내

발급 순서:
  (1) https://home.openweathermap.org/users/sign_up 가입
  (2) 이메일로 온 확인(Verify) 링크 클릭  ← 필수 (안 하면 401)
  (3) My API keys 페이지에서 Default 키 복사
      https://home.openweathermap.org/api_keys

주의: 새로 발급한 키는 활성화까지 최대 2시간 걸릴 수 있음

7. "튼구름" 표현 자연스럽게 변경
==========================================================
질문: "튼구름 이라고 뜨는데 왜 저렇게 뜨는거야?"

원인:
- 버그가 아니라 OpenWeatherMap 이 lang=kr 로 주는 공식 번역
- 영어 원문 'broken clouds'(코드 803) 를 직역한 표현
- OpenWeatherMap 구름 4단계:
    801 few clouds      → (원문) 약간의 구름이 낀 하늘
    802 scattered clouds→ (원문) 흩어진 구름
    803 broken clouds   → (원문) 튼구름    ← 어색
    804 overcast clouds → (원문) 온흐림

요청: "자연스럽게 변경해줘"
→ 날씨 코드(weather[0].id) 기준으로 자연스러운 한글 매핑 추가
  (WEATHER_KO 딕셔너리 + weatherKo() 함수, 없으면 API 원문으로 폴백)
  주요 변경:
    튼구름        → 구름 많음
    온흐림        → 흐림
    흩어진 구름   → 구름 조금
    + 비/눈/뇌우/안개/이슬비 등 전체를 자연스러운 표현으로 정리
      (약한 비, 소나기, 폭설, 진눈깨비, 황사 ...)

사용한 API 정리
==========================================================
[ 도시명 검색 ]
GET https://api.openweathermap.org/data/2.5/weather
  ?q={city}
  &appid={API_KEY}
  &units=metric      ← 섭씨
  &lang=kr           ← 한글 날씨 설명

[ 좌표(현재 위치) 검색 ]
GET https://api.openweathermap.org/data/2.5/weather
  ?lat={lat}&lon={lon}&appid={API_KEY}&units=metric&lang=kr

[ 날씨 아이콘 ]
https://openweathermap.org/img/wn/{icon}@2x.png

[ 주요 응답 필드 ]
- weather[0].description : 날씨 설명(한글)
- weather[0].icon        : 아이콘 코드 (끝 d=낮 / n=밤)
- main.temp / feels_like / temp_min / temp_max : 온도(°C)
- main.humidity / pressure : 습도(%) / 기압(hPa)
- wind.speed / deg / gust  : 풍속 / 풍향 / 돌풍
- visibility               : 가시거리(m)
- clouds.all               : 구름량(%)
- sys.sunrise / sunset     : 일출/일몰(UTC초)  *현지시각 = +timezone
- timezone                 : 해당 도시 UTC offset(초)


==========================================================
배운 점 / 핵심 포인트
==========================================================
- units=metric → 섭씨, lang=kr → 한글 설명
- API 키는 발급 후 활성화에 시간이 걸릴 수 있음(401 ≠ 잘못된 키일 수도)
- Geolocation API 로 사용자 위치 자동 감지(보너스 기능)
- API 키를 코드에 넣으면 편하지만 공개 저장소엔 노출 위험
  → .gitignore 처리로 보호
- lang=kr 번역이 어색할 수 있음(예: 튼구름)
  → 날씨 코드(weather[0].id) 기준으로 직접 한글 매핑하면 깔끔
- 단일 HTML 파일 + CDN React/Tailwind 로 빌드 도구 없이 완성
