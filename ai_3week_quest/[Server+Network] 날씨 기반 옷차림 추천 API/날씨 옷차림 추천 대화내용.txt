■ 1. 프로젝트 요청
────────────────────────────────────────
사용자:
  single-react-dev 와 single-server-specialist 두 에이전트가 함께
  [Server+Network] 날씨 기반 옷차림 추천 API를 제작해줘.
  1) 내 Node 서버(express)에 /recommend 같은 엔드포인트를 만들고
  2) 서버가 외부 날씨 API를 호출해 현재 기온·날씨를 받아온 뒤
  3) 기온 구간에 따라 옷차림 문구(예: "패딩 필수", "반팔 OK")를
     가공해서 응답하게 하기.
  ★ API 키는 프론트가 아니라 서버에 숨기기 (백엔드를 따로 두는 핵심 이유!)
  - 날씨 데이터는 OpenWeatherMap(무료 키) 사용
  - 프론트는 내 서버(/recommend)만 호출, 외부 API 키는 .env에 보관
  - '기온 구간 → 옷차림' 매핑 표를 만들어 로직을 깔끔하게

Claude:
  두 에이전트가 동일한 API 규약에 맞춰 충돌 없이 작업하도록
  공통 사양을 먼저 정의하고 병렬로 실행함.

  [공통 API 규약]
   - GET /recommend?city=서울
   - 응답: {
       city, temp, feelsLike, weatherDesc, icon, tempRange,
       recommendation: { headline, items[], emoji }
     }
   - 에러: { error: "한국어 메시지" } + 적절한 status code
   - 서버는 process.env.OPENWEATHER_API_KEY 사용(.env), index.html 정적 서빙

  [역할 분담]
   - single-server-specialist → server.js + package.json + .env.example
                                + .gitignore + README.md
       (Express, 정적 서빙, OpenWeatherMap 호출, 기온 구간 매핑,
        API 키 .env 보관)
   - single-react-dev → index.html
       (React CDN 단일 파일, 도시 입력 → /recommend 호출 → 결과 카드,
        외부 API/키는 일절 모름)

  ※ 두 에이전트 모두 백그라운드 모드에서 Write 권한 승인 창을 띄울 수
    없어 멈췄으나, 사양이 명확했기에 Claude가 직접 모든 파일을 작성하고
    실제 서버 구동·검증까지 완료함.

  [기온 구간 → 옷차림 매핑표] (server.js의 CLOTHING_TABLE)
    28℃ 이상  🥵 한여름! 시원하게      민소매, 반팔, 반바지, 린넨옷

    17~19℃    🍂 얇은 니트/가디건      가디건, 니트, 맨투맨, 청바지
    12~16℃    🧥 자켓 필요            자켓, 가디건, 야상, 스타킹, 청바지
    9~11℃     🌬️ 트렌치코트/자켓       트렌치코트, 자켓, 니트, 청바지, 스타킹
    5~8℃      🧣 코트+따뜻한 옷        코트, 히트텍, 니트, 두꺼운 바지, 목도리

  [완성 파일]
   - server.js      : Express 서버, GET /recommend → 지오코딩 + 날씨 조회
                      → 기온 구간 매핑 → 옷차림 JSON 응답
   - index.html     : React 단일 페이지(도시 입력/결과 카드/기온별 테마)
   - package.json   : express, dotenv 의존성, "npm start" 스크립트
   - .env.example   : OPENWEATHER_API_KEY 템플릿 + 발급 안내
   - .gitignore     : node_modules, .env 제외 (키 유출 방지)
   - README.md      : 키 발급 → 실행 → API 명세 → 매핑표

■ 3. API 키 발급 안내 & 입력
────────────────────────────────────────
사용자:
  api 받는 방법 알려줘.

Claude:
  OpenWeatherMap 무료 키 발급 절차 안내.
   1) openweathermap.org/api 가입 → 이메일 인증

  ※ 신규 키는 활성화까지 수십 분~2시간 걸릴 수 있음(401 시 잠시 대기).

사용자:
  (OpenWeatherMap API 키 입력)

Claude:
  키를 .env에 저장하고 실제 서버 구동으로 라이브 검증.

  [발견한 버그 & 수정]
   OpenWeatherMap의 q=날씨조회는 한글 도시명("서울")을 못 찾아
   기본값 검색이 404가 났음.
   → 지오코딩 API(/geo/1.0/direct)로 도시명(한글/영문)→위경도 변환 후
     좌표로 날씨를 조회하도록 수정. 한글 도시명 정상 동작 + 응답에
     한글 지명("부산광역시", "도쿄도")까지 표시.

  [검증 결과]
   서울    29.1℃ → 🥵 "28℃ 이상" (민소매/반팔/반바지/린넨옷)  200
   부산    23.4℃ → 😎 "23~27℃" (반팔 OK)                    200
   Tokyo   20.4℃ → 🙂 "20~22℃" (얇은 겉옷)                  200
   city 생략 → 기본 서울 정상                                200
   없는 도시 → "도시를 찾을 수 없습니다..."                   404
   ★ API 키는 .env에만 존재, 응답/프론트 어디에도 노출 없음

■ 5. 날씨 설명 자연스럽게 수정
────────────────────────────────────────
사용자:
  "튼구름" 같은 알맞지 않은 말이 나오는 걸 체크해서 변경해줘.

Claude:
  "튼구름"은 OpenWeatherMap의 lang=kr 자동번역이 어색하게 나오는
  사례(broken clouds=803). "온흐림", "실 비" 등도 마찬가지.
  → 외부 번역 의존을 버리고, 날씨 코드(weather[0].id) 기준으로
    자연스러운 한국어를 직접 매핑(WEATHER_DESC_BY_ID + 범위 fallback).

  [매핑 예시]
    803 튼구름 → 구름 많음
    804 온흐림 → 흐림
    802 구름조금 → 구름이 약간
    500 실 비 → 약한 비
    800 맑음 → 맑음

  [여러 도시 재검증]
    서울 → 구름 많음   부산 → 구름이 약간   London → 흐림
    Reykjavik → 흐림   Moscow → 약한 비
  화면에서 "튼구름" → "구름 많음" 으로 바뀐 것 시각 확인 완료.

■ 핵심 학습 포인트
==================================================
 1. API 키는 프론트가 아니라 서버 환경변수(.env)에 숨긴다.
    → 프론트는 내 서버(/recommend)만 호출하므로 키가 브라우저에
      절대 노출되지 않는다. (백엔드를 따로 두는 핵심 이유)
 2. .env 는 .gitignore에 등록해 깃에 올라가지 않게 한다(키 유출 방지).
 3. '기온 구간 → 옷차림' 매핑을 데이터 표(CLOTHING_TABLE)로 분리하면
    구간/문구 수정이 쉽고 로직이 깔끔하다.
 4. 한글 도시명은 지오코딩 API로 좌표 변환 후 조회하면 안정적이다.
 5. 외부 API의 번역/표현이 어색하면, 코드(id)를 기준으로 우리가
    직접 한국어를 매핑해 사용자 경험을 다듬는다.

■ 실행 방법
==================================================
 1) npm install
 2) cp .env.example .env  후 .env에 OPENWEATHER_API_KEY 입력
 3) npm start             (포트 충돌 시: PORT=3999 npm start)
 4) 브라우저 → http://localhost:3000

■ 기술 스택
 - 백엔드: Node 18+(내장 fetch), Express, dotenv
 - 프론트: React 18 + Babel standalone (CDN, 빌드 도구 없음)
 - 외부 API: OpenWeatherMap Geocoding + Current Weather (units=metric, lang=kr)
