■ 진행 단계 (요약)
----------------------------------------------------------------

1) "나만의 ChatGPT 봇" 1차 제작 (규칙 기반)
   - 미션: AI 프로필(성격/말투/전문분야) 설정 → 그대로 답변 → 대화 누적 UI
   - 단일 index.html 안에서 React + Tailwind로 구현
   - 실제 LLM 대신 규칙 기반(키워드 매칭 + 말투 변환)으로 답변 생성

2) 진짜 ChatGPT(OpenAI) API 연동
   - 처음엔 브라우저에서 직접 OpenAI 호출 + API 키 입력 UI(localStorage)
   - 프로필을 system 프롬프트로 변환해 페르소나를 LLM에 지시
   - "똑같은 말만 반복" 현상 → 원인은 API 키 미연결(규칙 기반 폴백 상태)였음

3) 백엔드 분리 (보안)
   - 심리상담 프로젝트(server.js/server.py) 구조를 참고
   - 키를 브라우저에 노출하지 않도록 백엔드가 OpenAI를 "중계"
   - 계약: POST /api/chat  { messages, system } → { success, data:{ reply } }
   - index.html 은 자기 백엔드(/api/chat)만 호출하도록 변경
   - 포트 3457 사용 (심리상담 3456과 충돌 방지)
   - Node 미설치 환경이라 python3 server.py 로 구동 (server.py 추가 생성)

4) 키 이슈 점검
   - 가져온 OpenAI 키가 폐기(revoke)된 상태로 확인됨
     → 평문 키가 git 등 공개에 노출되면 OpenAI가 자동 폐기
   - 해결: 새 키를 환경변수 OPENAI_API_KEY 로 주입 (코드에 키 하드코딩 금지)

5) Claude(Anthropic) API 로 교체 → 다시 OpenAI 로 복귀
   - 잠깐 Claude Messages API(api.anthropic.com)로 바꿔봄
     · system은 최상위 필드, messages는 user로 시작해야 하는 차이 반영
   - "MAX 구독을 써도 API 키는 유료인가?" → 그렇다. 구독과 API는 별도 결제.
   - 최종적으로 사용자 요청에 따라 OpenAI(gpt-4o-mini) 버전으로 되돌림

6) 심심이(SimSimi) 스타일로 전면 리디자인
   - 요리/전문분야 컨셉 삭제 → 예전 유행하던 "심심이" 컨셉으로 변경
   - 노란 테마 + CSS로 그린 심심이 마스코트 얼굴(눈/볼터치/입, 까딱 애니메이션)
   - 성격(기분) 4종: 😜 장난꾸러기 / 😏 까칠이 / 🥰 다정이 / 🛸 4차원
   - 항상 반말로 짧고 위트있게 받아치는 수다 톤 (system 프롬프트)
   - 규칙 기반 폴백도 심심이식 드립으로 교체 (백엔드 실패 시에도 동작)

7) 버그 수정 — 한글 IME 조기 전송
   - 증상: 채팅 입력 중 글자를 확정하는 Enter가 곧바로 전송돼
           "다음 채팅이 자동으로 나오는" 현상
   - 원인: 한글 조합 확정용 Enter까지 전송으로 처리
   - 수정: keydown에서 e.nativeEvent.isComposing 이면 전송 무시
           (한글 조합이 끝난 뒤의 Enter에서만 전송)

■ 핵심 구조
----------------------------------------------------------------

[브라우저: index.html]
  - buildSystemPrompt(profile): 심심이 성격(기분)을 system 프롬프트로 변환
  - fetchBackendReply(): POST /api/chat 로 { messages, system } 전송
  - 실패 시 generateReply()로 심심이식 규칙 기반 폴백
  - Enter 전송(IME 조합 가드), Shift+Enter 줄바꿈, 자동 스크롤

[백엔드: server.py / server.js]
  - 정적 서빙: GET / , /index.html → index.html
  - POST /api/chat: 클라이언트 system + messages 를 OpenAI에 중계
  - 모델: gpt-4o-mini, 키는 환경변수 OPENAI_API_KEY 로만 주입


----------------------------------------------------------------
■ 배운 점 / 주의사항
----------------------------------------------------------------
  - API 키는 절대 프런트(브라우저)나 코드에 평문으로 두지 말 것
    → 백엔드가 관리 + 환경변수로 주입
  - 공개 저장소에 키가 올라가면 자동 폐기될 수 있음
  - 구독(Max/Pro)과 개발자 API는 결제가 별개 (API는 종량제)
  - 한글 입력 폼은 IME 조합(isComposing) 처리를 꼭 신경 쓸 것
