## 3. 2주차 — 순수 프론트엔드 도구 (HTML 한 파일)

- 핵심 개념: 단일 index.html 파일 안에 HTML(구조)·CSS(스타일)·JavaScript(동작)를 모두 담는다.
- 빌드 도구나 서버 없이 브라우저로 파일을 바로 열어 실행한다.
- 외부 기능이 필요하면 CDN script 태그로 라이브러리를 불러온다. npm install이 필요 없다.
- 디자인은 토스(Toss) 스타일 같은 깔끔한 모바일 UI를 자주 참고했다.
- 만든 것: 더치페이 계산기 — 총액과 인원을 입력하면 1/n 정산 금액을 계산한다.
- 만든 것: 세금 계산기 — 금액을 입력하면 부가세·세금을 계산한다.
- 만든 것: QR 코드 생성기 — 텍스트나 URL을 입력하면 QR 이미지를 생성한다(qrcode 라이브러리 사용).
- 만든 것: PDF 생성기 — 내용을 PDF로 변환한다(jsPDF 라이브러리 사용).

- 기억할 패턴: getElementById로 입력값을 읽고, 계산 후 결과 영역의 textContent나 innerHTML을 바꾼다.
- 기억할 패턴: 버튼에 onclick이나 addEventListener('click', ...)로 동작을 연결한다.

## 4. 3주차 스터디 — 외부 API 연동 (네트워크)

- 핵심 개념: 브라우저에서 fetch(url)로 외부 서버(API)에 요청을 보내고 JSON 데이터를 받아 화면에 그린다.
- 비동기 처리는 async/await로 한다. fetch는 Promise를 반환하므로 await fetch 다음에 await res.json() 순서로 데이터를 꺼낸다.
- 응답 상태 확인: res.ok가 false면(404, 401 등) 에러로 처리한다.
- 만든 것: NASA 오늘의 우주 사진(APOD API) — NASA API에서 매일의 사진과 설명을 받아 보여준다.
- 만든 것: 날씨앱(OpenWeatherMap API) — 도시명을 입력하면 현재 날씨를 받아 아이폰 날씨앱 스타일로 표시한다.
- 만든 것: 포켓몬 도감(PokeAPI) — pokeapi.co에서 포켓몬 정보와 이미지를 받아 도감처럼 표시한다.
- 포켓몬 도감에서 "하드코딩 5마리" 버전과 "API 호출" 버전을 비교하며 API의 장점을 배웠다.
- 비교 학습: 데이터를 코드에 직접 박아두면(하드코딩) 늘리기 어렵지만, API를 쓰면 같은 코드로 수많은 데이터를 다룰 수 있다.
- 만든 것: 심리상담 채팅 — 채팅 UI에 서버를 연동했다.
- API 키가 필요한 서비스는 가입 후 키를 발급받아야 하고, 신규 키는 활성화에 시간이 걸릴 수 있다(401이면 잠시 후 재시도).

## 5. 3주차 퀘스트 — 백엔드 서버 + AI (가장 중요한 개념)

- 왜 서버가 필요한가: 외부 유료 API(OpenAI, Anthropic, OpenWeatherMap 등)의 API 키를 브라우저(프론트엔드)에 두면 안 된다.
- 이유: 누구나 브라우저 개발자 도구로 프론트엔드에 있는 키를 훔쳐볼 수 있기 때문이다.
- 그래서 내 서버를 한 단계 둔다: 브라우저 index.html → 내 서버 /api/... → 외부 API(OpenAI 등) 순서로 호출한다.
- 프론트는 오직 내 서버만 호출하고, 키는 서버의 환경변수(.env)에만 둔다. 그래서 브라우저에 키가 절대 노출되지 않는다.
- 이것이 백엔드를 따로 두는 핵심 이유다.
- 서버 기술 스택: Node.js 18 이상을 쓰며, 보통 Express.js 프레임워크로 서버를 작성한다.
- 서버의 3가지 일: 정적 파일 서빙(index.html 제공), JSON 본문 파싱(프론트가 보낸 POST 데이터 읽기), API 엔드포인트(외부 API 호출 후 결과를 JSON으로 반환).
- API 키는 process.env.ANTHROPIC_API_KEY 처럼 환경변수로 읽고, 없으면 친절한 한국어 에러 메시지를 반환한다.
- AI API 호출(Anthropic Claude): POST api.anthropic.com/v1/messages, 헤더에 x-api-key와 anthropic-version 2023-06-01, system 프롬프트와 messages 배열을 보낸다.
- AI API 호출(OpenAI): POST api.openai.com/v1/chat/completions, 헤더에 Authorization Bearer 키, messages에 system/user 역할로 프롬프트를 전달한다.
- 시스템 프롬프트로 AI의 역할과 규칙을 정한다. 예: "이 문서 내용만 근거로 답하고 없으면 몰라요라고 해".
- RAG의 기초: 참고 문서를 통째로 시스템 프롬프트에 넣고 "이 내용만 근거로 답하라"고 지시하면, 환각(없는 내용을 지어내기)을 막을 수 있다.
- 이 "나를 설명하는 Q&A 앱"도 바로 그 RAG 기초 패턴으로 만들어졌다 — about-me.md를 근거로만 답한다.
- 기억할 패턴: 에러는 항상 { "error": "한국어 메시지" } 형태로 내려 사용자에게 친절히 안내한다.
- 기억할 패턴: 외부 API 결과를 그대로 주지 않고 서버에서 한 번 가공해 깔끔한 JSON으로 내려주면 프론트가 단순해진다.
