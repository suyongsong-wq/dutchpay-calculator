==========================================================
[Context + Server + AI] 나를 설명하는 .md + Q&A 앱 — 제작 대화내용
하버스쿨 AI 교육 · 3주차 퀘스트
==========================================================

■ 미션
  1) 나의 이력·취향·경험을 정리한 about-me.md 컨텍스트 파일을 만들고
  2) 사용자가 "이 사람 무슨 일 해?" 같은 질문을 입력하면,
     내 서버가 [질문 + about-me.md]를 함께 AI에게 전달하고
  3) AI가 about-me.md 내용을 근거로 답변 (없는 내용은 "몰라요"라고 답)

■ 이 앱의 핵심 (RAG의 기초)
  - AI의 일반 지식이 아니라 '내 about-me.md'를 근거로만 답하게 하는 것이 핵심.
  - 서버가 시스템 프롬프트에 about-me.md 전체를 <about-me>...</about-me> 로 실어 보내고,
    "이 내용에 있는 것만 근거로 답하고, 없으면 몰라요라고 해"라고 지시 → 환각 방지.
  - about-me.md 한 파일만 갈아끼우면 '친구 소개 봇', '우리 가게 소개 봇'으로 확장 가능.

==========================================================
■ 제작 과정 (요약)
==========================================================

1. 두 에이전트 협업으로 골격 제작
   - single-server-specialist → server.js (백엔드)
   - single-react-dev        → index.html (프론트엔드)
   - 고정 API 계약: POST /api/ask  { question } → { answer }

2. "키 없이도 돌아가게" — 우리 실제 수업 내용 기반 MD 작성
   - 레포의 실제 커리큘럼(수업노트)을 바탕으로 about-me.md를 작성.
   - 한 줄에 하나의 사실(키워드 밀도 높게)로 적어, API 키가 없어도
     서버가 about-me.md에서 직접 관련 줄을 찾아 답하는 '로컬 검색 모드'가 잘 동작하도록 함.

3. 디자인을 텔레그램 스타일로 변경
   - 파란 헤더 바(아바타+제목+온라인), 연블루 도트 패턴 배경,
     보낸 메시지=연녹색 말풍선(오른쪽,꼬리+시각+✓✓), 받은 메시지=흰색 말풍선(왼쪽),
     하단 둥근 입력바 + 원형 종이비행기 전송 버튼.

■ 파일 구성
==========================================================
  about-me.md   — 나와 내가 배운 수업 내용(컨텍스트=지식의 전부). 섹션: 기본정보/커리큘럼/2주차/3주차/프로젝트/취향
  server.js     — 질문+about-me.md를 근거로 답. 키 없으면 로컬 검색, 키 있으면 claude-opus-4-8. 외부 의존성 0개(Node 내장만)
  index.html    — React+Tailwind(CDN) 텔레그램 스타일 채팅 UI. /api/ask 연동
  package.json  — 실행 스크립트(start: node server.js)
  .env.example  — ANTHROPIC_API_KEY 설정 예시
  README.md     — 설명/실행법
  실행화면_1_대화 (텔레그램).png      — 일반 질문 답변 화면
  실행화면_2_그라운딩 (텔레그램).png  — 프로젝트 질문 + 문서에 없는 질문 처리 화면

==========================================================
■ 동작 방식
==========================================================
  [브라우저 index.html]  --POST /api/ask {question}-->  [내 서버 server.js]
        키 없음(프론트)                                    │
                                                           ├─(키 있음) about-me.md를 system 프롬프트에 실어 claude-opus-4-8 호출
                                                           └─(키 없음) about-me.md 본문에서 질문 키워드로 관련 줄 검색
                                                           │
  [화면에 답변 표시]  <----------- { answer } -------------┘   (근거 없으면 "몰라요")

  ※ API 키는 서버에만 둔다(process.env.ANTHROPIC_API_KEY). 프론트는 /api/ask만 호출 → 키가 브라우저에 노출되지 않음.

==========================================================
■ 실행 방법
==========================================================
  # 키 없이 (about-me.md 로컬 검색 모드 — 키 없이도 잘 돌아감)
  node server.js

  # 키와 함께 (Claude로 더 자연스러운 답)
  ANTHROPIC_API_KEY=sk-ant-... node server.js

  # 브라우저에서 http://localhost:3000 접속 (포트 사용 중이면 PORT=4500 node server.js)

Q. 무슨 일 해?
A. · 한 줄 소개: 하버스쿨 AI 교육 과정에서 직접 만들어 보며 배우는 실용주의 학습자
   · 지금 하는 일: 하버스쿨 AI 교육 과정 수강 중 (현재 3주차)
   · 배우는 방식: "개념만 듣기보다 작동하는 결과물을 직접 손으로 만들면서 배운다"

Q. 왜 API 키를 서버에 숨겨?
A. · 3주차 퀘스트에서는 백엔드 서버와 AI를 배웠다: Node.js 서버를 두고 API 키를 숨기며 AI API까지 호출했다.
   · 왜 서버가 필요한가: 외부 유료 API(OpenAI, Anthropic, OpenWeatherMap 등)의 API 키를 브라우저(프론트엔드)에 두면 안 된다.
   · 프론트는 오직 내 서버만 호출하고, 키는 서버의 환경변수(.env)에만 둔다. 그래서 브라우저에 키가 절대 노출되지 않는다.

Q. 어떤 프로젝트 만들었어?
A. · 3주차 퀘스트: AI 별명 생성기, AI 꿈해몽 앱, 나를 소개하는 Q&A 봇, My ChatGPT, AI 이미지 생성기, 실시간 코인 시세 대시보드, 날씨 기반 옷차림 추천 API.
   · 3주차 스터디: NASA 오늘의 우주 사진, 날씨앱, 포켓몬 도감, 심리상담 채팅.
   · 2주차: 더치페이 계산기, 세금 계산기, QR 코드 생성기, PDF 생성기, 짤 생성기(밈 메이커).

Q. 좋아하는 음식은?  (about-me.md에 없는 정보 → 관련 취향만 안내/없으면 "몰라요")
A. · 좋아하는 것: 새로운 도구를 직접 만들어 보는 것, 복잡한 걸 깔끔하게 정리해 보여주는 것, 좋은 디자인과 매끄러운 사용자 경험.

  · 빈 질문 전송 → 400 "질문을 입력해 주세요."
  · GET / → 200 (채팅 화면 정상 서빙)
