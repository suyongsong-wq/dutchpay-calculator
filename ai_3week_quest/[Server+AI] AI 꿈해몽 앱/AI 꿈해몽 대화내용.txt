====================================================================
 [Server+AI] AI 꿈해몽 앱 — 제작 대화 정리
 작성일: 2026-06-15
====================================================================

■ 한 줄 소개
  어젯밤 꿈을 입력하면 AI(또는 서버 자체 꿈사전)가 해몽해 주는 앱.
  해몽가 캐릭터(페르소나)를 고르고, 결과를 "한줄요약 → 상징 키워드 →
  길흉 판정 → 오늘의 조언 → 행운지수 100점"으로 보여준다.
  디자인은 '점신' 앱의 '오늘의 운세' 화면 스타일.


--------------------------------------------------------------------
1. 최초 요청
--------------------------------------------------------------------
  "Server + AI를 조합해서 어젯밤 꿈을 입력하면 풀이해주는 AI 꿈해몽 앱을
   만들어라. 두 에이전트(single-react-dev + single-server-specialist)가
   함께 작업."

  핵심 요구사항
   1) 해몽가 AI의 캐릭터(말투·관점)를 시스템 프롬프트로 설정
   2) 사용자가 꿈을 입력하면 내 서버가 AI를 호출해 해몽 결과 생성
   3) 길몽/흉몽 + 오늘의 한 줄 조언까지 화면에 표시
   4) 출력 형식 고정: 1)한줄요약 2)상징 키워드 3)길몽/흉몽 4)오늘의 조언
   5) 재미 요소로 '행운지수 100점' 추가
   6) 캡처해서 공유하고 싶은 예쁜 카드 UI (바이럴 포인트)


--------------------------------------------------------------------
2. 두 에이전트 협업 — 공통 API 계약 정의
--------------------------------------------------------------------
  충돌을 막기 위해 먼저 "공통 계약"과 "파일 소유권"을 정하고 병렬 실행.

  · 엔드포인트 : POST /api/interpret
  · 요청       : { dream: string, persona: "mystic"|"mz"|"monk"|"bestie" }
  · 응답       : { summary, keywords[], verdict, advice, luckScore }
                 - verdict 는 "길몽" | "흉몽" | "반길몽반흉몽"
                 - luckScore 는 0~100 정수
  · 파일 소유권: server.js / package.json → 서버 전문가
                 index.html               → 리액트 개발자

  ※ 기존 형제 프로젝트 '[Server+AI] AI 별명 생성기'의 패턴
    (Express + OpenAI gpt-4o-mini + 내장 fetch + JSON 응답)을 그대로 계승.

  ※ 두 에이전트가 새 폴더 생성 권한에 막혀, 설계안을 받아
    최종 파일은 직접 작성·검증함.


--------------------------------------------------------------------
3. 해몽가 페르소나 (시스템 프롬프트로 말투/관점 주입)
--------------------------------------------------------------------
  🔮 mystic  신비로운 점술가 — 예스럽고 신비로운 어투("~하리라"), 별·운명
  😎 mz      MZ 해몽러        — 요즘 말투 + 이모지 + 가벼운 드립
  🧘 monk    해탈한 스님       — 담담한 선문답풍, 집착을 내려놓는 관점
  🫶 bestie  다정한 친구       — 따뜻하게 공감하고 다독여주는 말투


--------------------------------------------------------------------
4. 디자인 변경 — '점신' 오늘의 운세 스타일
--------------------------------------------------------------------
  사용자가 점신 앱 스크린샷을 공유 → 그 톤으로 전면 재디자인.

  · 밝은 흰색/연회색 배경, 모바일 폰 레이아웃(max 420px)
  · 상단 앱바(‹ 뒤로 · 가운데 제목 · 공유 ↗)
  · 부드러운 하늘색 물결(wave) 헤더 카드
  · 큰 점수 숫자 + 위에 노란 형광펜 점 (점신의 "85" 연출)
  · 검은 알약(pill) 탭으로 해몽가 선택
  · 노란색 CTA 버튼 ("운세 풀어 보기" 스타일)


--------------------------------------------------------------------
5. 두 가지 엔진 — AI 버전 / 내부 서버 버전 토글
--------------------------------------------------------------------
  화면 상단 세그먼트 토글로 전환.

  🤖 AI 버전     → POST /api/interpret
                  실제 OpenAI gpt-4o-mini 호출 (OPENAI_API_KEY 필요)

  ⚙️ 내부 서버    → POST /api/interpret-local
                  API 키 없이 서버 자체 '꿈사전(룰 기반)'으로 해몽.
                  꿈 텍스트에서 상징 추출 → 행운지수·길흉 계산 →
                  페르소나 말투로 요약/조언 생성. 항상 동작(데모용).

  결과 카드 헤더에 어떤 엔진으로 풀었는지 뱃지(🤖 AI / ⚙️ 내부 서버) 표시.


--------------------------------------------------------------------
6. 꿈사전 모듈 분리 + 상징 추가
--------------------------------------------------------------------
  · 룰 사전을 별도 모듈 dream-dictionary.js 로 분리 (총 77개 상징)
    카테고리: 동물 / 자연 / 사람 / 사물 / 음식 / 신체 / 행동
    각 항목: match(찾을 키워드), kw(상징명), category, luck(가감점),
             verdict(길흉), meaning(뜻)
  · GET /api/dictionary?q=검색어&category=동물  — 사전 검색·조회 API
  · index.html 하단에 '꿈 사전' 탭 추가 (키워드 검색 + 카테고리 필터)
  · 요청으로 '돈다발' 상징 추가 (물고기·거미는 이미 존재)
      돈다발 / 사물 / 길몽 / luck +24 /
      "큰 재물·횡재수·뜻밖의 수입이 들어올 강력한 길몽이에요."

  ▶ 사전 항목을 더 추가하려면 dream-dictionary.js 의
    DREAM_SYMBOLS 배열에 객체를 하나 넣으면 됨 (검색·해몽 자동 반영).


--------------------------------------------------------------------
7. 브라우저 동작 확인 (헤드리스 Chrome 캡처)
--------------------------------------------------------------------
  · 입력 화면      → 정상 (앱바·물결·토글·페르소나·CTA·하단 탭)
  · "길에서 돈다발을 주웠는데 하늘을 날아올랐어요" (MZ, 내부 서버)
       → 길몽 · 행운지수 99점 · 키워드 #돈다발 #재물 #비상
  · 꿈 사전에서 '돈다발' 검색 → 길몽 카드 + 태그 정상 노출


--------------------------------------------------------------------
8. 파일 구조
--------------------------------------------------------------------
  [Server+AI] AI 꿈해몽 앱/
   ├─ server.js                 Express 서버 (AI 호출 + 내부 해몽 + 사전 API)
   ├─ dream-dictionary.js       꿈사전 데이터·검색 로직 (77개 상징)
   ├─ index.html                React(CDN) 프론트엔드 (점신 스타일 UI)
   ├─ package.json              express 의존성
   ├─ AI 꿈해몽 대화내용.txt      (이 문서)
   └─ 실행화면_*.png             실행 스크린샷


--------------------------------------------------------------------
9. 실행 방법
--------------------------------------------------------------------
   1) 의존성 설치 :  npm install
   2) (AI 버전만)  :  export OPENAI_API_KEY=sk-...
   3) 서버 실행    :  npm start   (또는 node server.js)
   4) 접속         :  http://localhost:3000

   ※ '내부 서버' 모드는 API 키 없이 바로 동작.
   ※ '🤖 AI 버전' 모드는 OPENAI_API_KEY 설정 후 실제 AI 해몽.
