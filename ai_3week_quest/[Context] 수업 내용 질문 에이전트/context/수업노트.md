# 하버스쿨 AI 교육 — 수업 노트 (텍스트 컨텍스트)

> 이 문서는 "수업 내용 질문 에이전트"가 답변의 근거로 삼는 **강의 노트**입니다.
> 코드 컨텍스트(`context/code/` 폴더의 실습 파일)와 함께 사용됩니다.
> 에이전트는 이 노트에 적힌 내용을 우선 근거로 답하고, 노트에 없으면 코드 컨텍스트를 참고하며,
> 둘 다 없으면 "수업 자료에는 안 나와요"라고 솔직하게 답합니다.

---

## 0. 전체 커리큘럼 한눈에 보기

| 주차 | 주제 | 핵심 기술 | 만든 것 |
|------|------|-----------|---------|
| 2주차 | **순수 프론트엔드 도구** | HTML/CSS/JS 단일 파일, CDN 라이브러리 | 계산기·변환기 시리즈 |
| 3주차(스터디) | **외부 API 연동(네트워크)** | `fetch`, async/await, JSON | NASA APOD, 날씨앱, 포켓몬 도감, 심리상담 챗 |
| 3주차(퀘스트) | **백엔드 서버 + AI/네트워크** | Node.js, Express, API 키 숨기기, AI API 호출 | AI 별명/꿈해몽/Q&A 봇, 코인 대시보드, ChatGPT 클론 |

큰 흐름: **(1) 브라우저만으로 만드는 도구 → (2) 브라우저에서 외부 API 직접 호출 → (3) 서버를 두고 키를 숨기며 AI까지 호출**.
난이도와 "왜 서버가 필요한가"에 대한 이해가 단계적으로 올라가는 구조입니다.

## 1. 2주차 — 순수 프론트엔드 도구 (HTML 한 파일)

### 1.1 핵심 개념
- **단일 `index.html` 파일** 안에 HTML(구조) + CSS(스타일) + JavaScript(동작)를 모두 담는다.
- 빌드 도구·서버 없이 브라우저로 파일을 바로 열어 실행한다.
- 외부 기능이 필요하면 **CDN `<script>`** 태그로 라이브러리를 불러온다. (`npm install` 불필요)
- 디자인은 "토스(Toss) 스타일" 같은 깔끔한 모바일 UI를 자주 참고했다.

### 1.2 만든 도구들
- **[계산] 더치페이 계산기**: 총액과 인원을 입력 → 1/n 정산 금액 계산. (코드: `context/code/[2주차][계산] 더치페이 계산기.html`)
- **[계산] 세금 계산기**: 금액 입력 → 부가세/세금 계산.
- **[변환] QR 코드 생성기**: 텍스트/URL 입력 → QR 이미지 생성. `qrcode.js` 같은 CDN 라이브러리 사용. (코드: `context/code/[2주차][변환] QR 코드 생성기.html`)
- **[변환] PDF 생성기**: 내용을 PDF로 변환. `jsPDF` 류 라이브러리 사용.
- **[변환] 짤 생성기(밈 메이커)**: 이미지 위에 텍스트를 얹어 밈 이미지 생성. `<canvas>` 활용.

### 1.3 기억할 패턴
- `document.getElementById(...)`로 입력값을 읽고, 계산 후 결과 영역의 `textContent`/`innerHTML`을 바꾼다.
- 버튼 `onclick` 또는 `addEventListener('click', ...)`로 동작을 연결한다.
- 라이브러리는 `<head>`나 `<body>` 끝에 CDN `<script src="...">`로 추가한 뒤, 그 라이브러리의 전역 객체(예: `QRCode`, `jspdf`)를 호출한다.

## 2. 3주차 (스터디) — 외부 API 연동 (네트워크)

### 2.1 핵심 개념
- 브라우저에서 **`fetch(url)`**로 외부 서버(API)에 요청을 보내고 **JSON 데이터**를 받아 화면에 그린다.
- 비동기 처리: **`async / await`**. `fetch`는 Promise를 반환하므로 `await fetch(...)` → `await res.json()` 순서로 데이터를 꺼낸다.
- 응답 상태 확인: `res.ok`가 false면(예: 404, 401) 에러 처리.

```js
async function getData() {
  try {
    const res = await fetch('https://api.example.com/items');
    if (!res.ok) throw new Error('요청 실패: ' + res.status);
    const data = await res.json();
    render(data);           // 받은 데이터를 화면에 그림
  } catch (e) {
    showError(e.message);   // 사용자에게 에러 안내
  }
}
```

### 2.2 만든 것들
- **NASA 오늘의 우주 사진 (APOD API)**: NASA API에서 매일의 사진+설명을 받아 보여줌.
- **날씨앱 (OpenWeatherMap API)**: 도시명을 입력 → 현재 날씨를 받아 표시 (아이폰 날씨앱 스타일 UI). (코드: `context/code/[3주차][네트워크] 날씨앱.html`)
- **포켓몬 도감 (PokeAPI)**: `https://pokeapi.co`에서 포켓몬 정보/이미지를 받아 도감처럼 표시. "하드코딩 5마리" 버전과 "API 호출" 버전을 비교하며 API의 장점을 학습. (코드: `context/code/[3주차][네트워크] 포켓몬 도감 (API).html`)
- **심리상담 채팅**: 채팅 UI + 서버 연동(`server.js` / `server.py`).

### 2.3 기억할 패턴 / 비교 학습
- **하드코딩 vs API**: 데이터를 코드에 직접 박아두면(하드코딩) 늘리기 어렵지만, API를 쓰면 같은 코드로 수많은 데이터를 다룰 수 있다 → 포켓몬 도감에서 직접 비교.
- API 키가 필요한 서비스(OpenWeatherMap 등)는 가입 후 키를 발급받아야 하며, 신규 키는 활성화에 시간이 걸릴 수 있다(401 시 잠시 후 재시도).

## 3. 3주차 (퀘스트) — 백엔드 서버 + AI/네트워크

### 3.1 왜 서버가 필요한가 (가장 중요한 개념)
- 외부 유료 API(OpenAI, Anthropic, OpenWeatherMap 등)의 **API 키를 브라우저(프론트엔드)에 두면 안 된다.** 누구나 개발자 도구로 키를 훔쳐볼 수 있기 때문.
- 그래서 **내 서버를 한 단계 둔다**:
  ```
  [브라우저 index.html] → [내 서버 /api/...] → [외부 API(OpenAI 등)]
        프론트(키 없음)        백엔드(키 숨김, .env)      외부 서비스
  ```
- 프론트는 **오직 내 서버**만 호출하고, 키는 서버의 **환경변수(`.env`)**에만 둔다. → 브라우저에 키가 절대 노출되지 않음. 이것이 백엔드를 따로 두는 핵심 이유.

### 3.2 서버 기술 스택 (Node.js)
- **Node.js 18+** (내장 `fetch` 사용 가능).
- **Express.js** 프레임워크로 서버 작성 (`npm install express`, 실행은 `npm start` 또는 `node server.js`).
- 서버의 3가지 일:
  1. **정적 파일 서빙**: `app.use(express.static(__dirname))`로 같은 폴더의 `index.html`을 브라우저에 제공.
  2. **JSON 본문 파싱**: `app.use(express.json())`로 프론트가 보낸 `POST` 데이터를 읽음.
  3. **API 엔드포인트**: `app.post('/api/...')`에서 외부 API를 호출하고 결과를 JSON으로 돌려줌.
- API 키는 `process.env.OPENAI_API_KEY` / `process.env.ANTHROPIC_API_KEY`처럼 환경변수로 읽는다. 없으면 친절한 한국어 에러 메시지를 반환.

```js
const express = require('express');
const app = express();
app.use(express.json());
app.use(express.static(__dirname));     // index.html 제공

app.post('/api/generate', async (req, res) => {
  const apiKey = process.env.OPENAI_API_KEY;
  if (!apiKey) return res.status(500).json({ error: 'API 키가 설정되지 않았어요.' });
  const { name } = req.body || {};
  // ... 외부 AI API를 fetch로 호출하고 결과를 res.json(...)으로 반환
});

app.listen(3000, () => console.log('http://localhost:3000'));
```

### 3.3 AI API 호출 패턴
- **OpenAI**: `POST https://api.openai.com/v1/chat/completions`, 헤더 `Authorization: Bearer <키>`, 모델 예: `gpt-4o-mini`. `messages`에 `system`/`user` 역할로 프롬프트 전달. (코드: `context/code/[3주차][서버+AI] AI 별명 생성기 server.js`)
- **Anthropic(Claude)**: `POST https://api.anthropic.com/v1/messages`, 헤더 `x-api-key: <키>` + `anthropic-version: 2023-06-01`. `system` 프롬프트와 `messages` 배열 전달. (코드: `context/code/[3주차][서버+AI] Q&A 봇 server.js`)
- **시스템 프롬프트로 역할/규칙을 정한다**: 예) "너는 별명 짓는 전문가야", "이 문서 내용만 근거로 답하고 없으면 '몰라요'라고 해".

### 3.4 만든 것들
- **[서버+AI] AI 별명 생성기**: 이름·성격·취미·원하는 톤을 입력 → AI가 재치있는 별명 5개를 이유와 함께 생성(JSON 형식 요청). (코드 참고)
- **[서버+AI] AI 꿈해몽 앱**: 꿈 내용을 입력 → AI가 해몽.
- **[서버+AI] 나를 소개하는 Q&A 봇**: `나에_대한_추측.md` 문서를 **유일한 근거**로 질문에 답하는 챗봇. 문서에 없는 건 "몰라요". 문서를 시스템 프롬프트의 `<about-me>...</about-me>`에 주입하는 방식. → **이번 "수업 내용 질문 에이전트"와 가장 비슷한 구조!** (코드 참고)
- **[네트워크] My ChatGPT**: ChatGPT 같은 채팅 클론.
- **[네트워크] AI 이미지 생성기**: 프롬프트 입력 → 이미지 생성.
- **[네트워크] 실시간 코인 시세 대시보드**: 코인 시세 API를 주기적으로 받아 대시보드로 표시.
- **[서버+네트워크] 날씨 기반 옷차림 추천 API**: 도시 → 서버가 OpenWeatherMap에서 날씨를 받아 기온 구간별 옷차림 매핑표로 가공해 추천. 키는 서버 `.env`에 숨김.

### 3.5 기억할 패턴
- **"문서를 근거로 답하는 봇"** 패턴(RAG의 기초): 참고 문서를 통째로 시스템 프롬프트에 넣고, "이 내용만 근거로 답하라"고 지시 → 환각(없는 내용 지어내기) 방지.
- 외부 API 결과를 그대로 주지 않고 **서버에서 한 번 가공**해서 깔끔한 JSON으로 내려주면 프론트가 단순해진다(옷차림 추천처럼).
- 에러는 항상 `{ "error": "한국어 메시지" }` 형태로 내려 사용자에게 친절히 안내.

## 4. 이번 과제 — "수업 내용 질문 에이전트"

### 4.1 미션
수업에서 배운 내용을 질문하면 정확히 답하는 나만의 AI 에이전트를 만든다.
- **텍스트 컨텍스트**(이 `수업노트.md`)와 **코드 컨텍스트**(`context/code/`의 실습 파일)를 **모두 참조**.
- **메모리**를 활용해 이전 대화 맥락을 기억(예: "아까 물어본 거랑 연결해서 설명해줘").

### 4.2 이 프로젝트의 구조 (메타 — 에이전트가 자기 자신에 대해 답할 수 있도록)
```
[Context] 수업 내용 질문 에이전트/
├── server.js            # Express 서버: 컨텍스트 로딩 + 메모리 + Claude 호출
├── index.html           # React(CDN) 채팅 UI
├── package.json
├── .env.example         # ANTHROPIC_API_KEY 설정 예시
├── context/
│   ├── 수업노트.md       # ← 지금 이 파일 (텍스트 컨텍스트)
│   └── code/            # 실습 코드들 (코드 컨텍스트)
└── memory/              # 대화 세션 기록(메모리)이 저장되는 곳
```
- 동작: 서버가 시작될 때 `context/` 폴더의 모든 텍스트/코드 파일을 읽어 **시스템 프롬프트**에 주입한다.
- 프론트(`index.html`)는 `/api/chat`만 호출한다. Anthropic API 키는 서버 `.env`에만 둔다(2~3주차에서 배운 "키 숨기기" 원칙 그대로).
- 세션별 대화 기록을 서버가 보관(메모리)하여, 매 질문마다 이전 대화를 함께 모델에 보내 맥락을 잇는다.

### 4.3 예시 질문 (이 에이전트가 답할 수 있어야 하는 것들)
- "지난 수업에서 배운 API 호출 방식이 뭐였지?" → 이 노트 2.1 + 코드로 답변
- "그때 작성한 더치페이 계산기 코드 보여줘" → `context/code`의 해당 파일에서 찾아 답변
- "왜 API 키를 서버에 숨겨야 해?" → 3.1로 답변
- "아까 물어본 내용이랑 연결해서 설명해줘" → 메모리로 직전 대화 맥락 활용
