## 전체 커리큘럼은 어떻게 구성돼 있나요?
<!-- keywords: 커리큘럼, 전체, 개요, 흐름, 주차, 무엇, 배운, 정리, 요약, 목차 -->
수업은 3단계로 점점 올라가는 구조예요.

1. **2주차 — 순수 프론트엔드 도구**: HTML 한 파일(HTML+CSS+JS)로 계산기·변환기를 만들었어요. 서버 없이 브라우저로 바로 실행하고, 필요한 기능은 CDN 라이브러리로 불러왔습니다.
2. **3주차(스터디) — 외부 API 연동(네트워크)**: 브라우저에서 `fetch`로 외부 API를 호출해 JSON 데이터를 받아 화면에 그렸어요. (NASA APOD, 날씨앱, 포켓몬 도감, 심리상담 챗)
3. **3주차(퀘스트) — 백엔드 서버 + AI**: Node.js/Express 서버를 두고 API 키를 숨긴 뒤, 서버를 거쳐 AI까지 호출했어요. (AI 별명/꿈해몽/Q&A 봇, 코인 대시보드, ChatGPT 클론, 옷차림 추천)

핵심 흐름: **브라우저만으로 만드는 도구 → 브라우저에서 외부 API 직접 호출 → 서버를 두고 키를 숨기며 AI까지 호출.**

## 단일 HTML 파일로 만든다는 게 무슨 뜻인가요?
<!-- keywords: 단일, html, 한 파일, 파일 하나, cdn, 라이브러리, 빌드, npm, 구조 -->
하나의 `index.html` 안에 **HTML(구조) + CSS(스타일) + JavaScript(동작)** 를 모두 담는 방식이에요.
- 빌드 도구나 서버 없이 브라우저로 파일을 바로 열어 실행합니다.
- 외부 기능이 필요하면 `<script src="...">` 로 **CDN 라이브러리**를 불러옵니다(`npm install` 불필요).
- 패턴: `document.getElementById(...)`로 입력값을 읽고 → 계산 → 결과 영역의 `textContent`/`innerHTML`을 바꿉니다. 버튼은 `onclick` 또는 `addEventListener('click', ...)`로 연결합니다.

## fetch로 API를 호출하는 방법을 알려줘
<!-- keywords: fetch, api, 호출, 비동기, async, await, 네트워크, json, 요청, 데이터, 받아오기 -->
브라우저에서 `fetch(url)`로 외부 서버에 요청을 보내고 **JSON**을 받아 화면에 그려요. 비동기라서 `async/await`를 씁니다.

```js
async function getData() {
  try {
    const res = await fetch('https://api.example.com/items');
    if (!res.ok) throw new Error('요청 실패: ' + res.status); // 404, 401 등 에러 처리
    const data = await res.json();  // 응답을 JSON으로 변환
    render(data);                    // 받은 데이터를 화면에 그림
  } catch (e) {
    showError(e.message);            // 사용자에게 에러 안내
  }
}
```

순서: `await fetch(...)` → 응답 `res.ok` 확인 → `await res.json()`으로 데이터 꺼내기.

<!-- keywords: 하드코딩, api, 차이, 비교, 포켓몬, 데이터, 장점 -->

## 왜 API 키를 서버에 숨겨야 하나요?
<!-- keywords: api 키, apikey, 키, 보안, 숨기, 서버, 환경변수, env, 노출, 왜, 안전, 프론트 -->
외부 유료 API(OpenAI, Anthropic, OpenWeatherMap 등)의 **키를 브라우저(프론트엔드)에 두면 누구나 개발자 도구로 훔쳐볼 수 있기 때문**이에요.

그래서 **내 서버를 한 단계 둡니다**:
```
[브라우저 index.html] → [내 서버 /api/...] → [외부 API(OpenAI 등)]
      프론트(키 없음)        백엔드(키 숨김, .env)      외부 서비스
```
프론트는 **오직 내 서버**만 호출하고, 키는 서버의 **환경변수(`.env`)**에만 둡니다. → 브라우저에 키가 절대 노출되지 않아요. 이것이 백엔드를 따로 두는 핵심 이유입니다.

## Express 서버는 어떻게 구성하나요?
<!-- keywords: express, 서버, node, nodejs, 백엔드, 정적, static, 엔드포인트, 구조, 만들기, server.js -->
Node.js 18+ 와 Express로 만들고, 서버는 3가지 일을 해요.
1. **정적 파일 서빙**: `app.use(express.static(__dirname))` → 같은 폴더의 `index.html`을 브라우저에 제공
2. **JSON 본문 파싱**: `app.use(express.json())` → 프론트가 보낸 POST 데이터를 읽음
3. **API 엔드포인트**: `app.post('/api/...')`에서 외부 API를 호출하고 결과를 JSON으로 반환

```js
const express = require('express');
const app = express();
app.use(express.json());
app.use(express.static(__dirname));      // index.html 제공

app.post('/api/generate', async (req, res) => {
  const apiKey = process.env.OPENAI_API_KEY;   // 키는 환경변수에서
  if (!apiKey) return res.status(500).json({ error: 'API 키가 없어요.' });
  // ... 외부 AI API를 fetch로 호출하고 res.json(...)으로 반환
});
app.listen(3000, () => console.log('http://localhost:3000'));
```
실행: `npm install` → `npm start`(또는 `node server.js`).

## AI API(OpenAI / Anthropic)는 어떻게 호출하나요?
<!-- keywords: openai, anthropic, claude, gpt, ai, api, 호출, 모델, 프롬프트, messages, 채팅 -->
서버에서 `fetch`로 호출해요. 키는 환경변수로 읽습니다.
- **OpenAI**: `POST https://api.openai.com/v1/chat/completions`, 헤더 `Authorization: Bearer <키>`, 모델 예 `gpt-4o-mini`. `messages`에 `system`/`user` 역할로 프롬프트 전달.
- **Anthropic(Claude)**: `POST https://api.anthropic.com/v1/messages`, 헤더 `x-api-key: <키>` + `anthropic-version: 2023-06-01`. `system` 프롬프트와 `messages` 배열 전달.

**시스템 프롬프트로 역할/규칙을 정합니다.** 예: "너는 별명 짓는 전문가야", "이 문서 내용만 근거로 답하고 없으면 '몰라요'라고 해".

## 시스템 프롬프트가 뭐고, '문서를 근거로 답하는 봇'은 어떻게 만드나요?
<!-- keywords: 시스템, 프롬프트, system, 문서, 근거, rag, 환각, 할루시네이션, q&a, qa, 봇, 역할 -->
**시스템 프롬프트**는 AI에게 역할과 규칙을 미리 정해주는 지시문이에요.

수업의 "나를 소개하는 Q&A 봇"은 **참고 문서를 통째로 시스템 프롬프트에 넣고** "이 내용만 근거로 답하고, 없으면 '몰라요'라고 해"라고 지시했어요. 이렇게 하면 AI가 없는 내용을 지어내는 **환각**을 막을 수 있죠. (이게 RAG의 기초예요.)

바로 이 "수업 내용 질문 에이전트"도 같은 구조예요 — 수업 자료(`수업노트.md`+코드)를 근거로 답합니다.

## 이번 과제(수업 내용 질문 에이전트)는 무엇이고 어떻게 동작하나요?
<!-- keywords: 이번, 과제, 에이전트, 질문, 컨텍스트, 메모리, 동작, 미션, 이 프로젝트, 너, 자기소개 -->
수업에서 배운 내용을 물어보면 **수업 자료를 근거로** 답하고 **이전 대화를 기억**하는 챗봇이에요.
- **텍스트 컨텍스트**(`수업노트.md`, 이 `지식베이스.md`)와 **코드 컨텍스트**(`context/code/`의 실습 파일)를 **모두 참조**합니다.
- **메모리**로 이전 대화를 기억해 "아까 물어본 거랑 연결해서" 같은 요청을 처리해요.
- 동작: 서버가 시작할 때 `context/` 폴더를 읽어둡니다. **API 키가 없으면** 이 지식베이스를 검색해 답하고(오프라인), **키가 있으면** Claude로 더 풍부하게 답합니다. 키는 서버에만 둡니다.
> 근거: 수업노트.md §4
